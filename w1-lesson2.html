🧾 Lesson 2: Static vs Dynamic Websites

🕒 Duration:

~30 minutes (Ideal for the second segment of Week 1)

🎯 Learning Objectives:

  • Understand what a static website is
  • Understand what a dynamic website is
  • Identify key differences between static and dynamic websites
  • Recognize when to use each type

🧩 Topics Covered:

📌 2.1 What is a Static Website?

A static website is made up of fixed files – mainly HTML, CSS, and possibly JavaScript – that are delivered exactly as they are written.

Characteristics:

  • Pre-built pages
  • Same content for every visitor
  • No server-side processing
  • Quick and easy to build
  • Fast loading (less server load)

Examples:

  • Personal portfolio
  • Company "About Us" page
  • Documentation pages
  • Landing pages

🔄 2.2 What is a Dynamic Website?

A dynamic website generates content on-the-fly based on user interaction, databases, or external inputs. It uses server-side languages and often connects to a database.

Characteristics:

  • Pages are generated dynamically
  • Content can change per user or over time
  • Uses back-end languages and databases
  • More complex and flexible

Examples:

  • Facebook, Twitter
  • E-commerce stores (e.g., Amazon)
  • Online booking systems
  • News sites with real-time updates

🔍 2.3 Key Differences Table:

Feature Static Website Dynamic Website
Content Fixed Changes based on interaction
Technologies HTML, CSS, JS HTML, CSS, JS + backend (PHP, Python, etc.)
Server-side logic ❌ None ✅ Yes
Database ❌ Not needed ✅ Usually required
Speed ⚡ Very fast 🐢 Slightly slower (depends)
Hosting Simple and cheap Requires more resources
Best for Portfolios, blogs, docs Social media, dashboards, e-commerce

🧪 Suggested Activities:

✅ Activity 1: Website Comparison

  • Show two websites (one static, one dynamic)
  • Ask students to compare features:
    • Does it show the same content to everyone?
    • Can users log in or interact?
    • Is data changing in real time?

Examples:

  • Static: getbootstrap.com
  • Dynamic: twitter.com

✅ Activity 2: Classify the Website

Give a list of website types. Ask students to classify each as static or dynamic:

  • Portfolio site → Static
  • Online store → Dynamic
  • Company landing page → Static
  • News website with login → Dynamic
  • Documentation site → Static

📝 Key Takeaways:

  • Static websites are ideal for simple, fast-loading pages with fixed content.
  • Dynamic websites allow user interaction, personalization, and real-time content.
  • Static websites are the foundation of web development and perfect for beginners.

📁 Optional Bonus:

Briefly introduce the idea of Static Site Generators (e.g., Jekyll, Hugo, Eleventy) for advanced learners. These tools generate static pages dynamically at build time — offering flexibility and performance without server-side complexity.
